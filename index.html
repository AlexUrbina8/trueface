01 · Problema

Identidades falsas en plataformas digitales.

Los deepfakes permiten crear rostros e identidades falsas cada vez más realistas, lo que representa un riesgo para la seguridad digital.

Riesgo principal

Un perfil falso puede usar una imagen generada por IA para hacerse pasar por una persona real.

Respuesta del proyecto

TrueFace funciona como una barrera defensiva capaz de detectar patrones visuales invisibles al ojo humano.

02 · Objetivo

Clasificar automáticamente imágenes faciales.

El objetivo es desarrollar un sistema capaz de clasificar imágenes faciales como reales o deepfake, con un flujo simple y aplicable en contextos de validación de identidad.

Ejemplo: Imagen de entrada → modelo → resultado: Real o Fake
DF

¿Qué es un deepfake?

Un deepfake es una imagen, video o audio manipulado con redes neuronales para simular rostros, voces o gestos reales.

03 · Dataset utilizado

Deepfake-vs-Real-60K.

Se utilizó un dataset público con imágenes reales y falsas, seleccionado por su alta calidad y por su enfoque en detección facial sintética, un problema clave de seguridad digital.

0imágenes de alta resolución
50/50distribución Real / Fake
0/10 = Fake · 1 = Real

Balanceado

El dataset contiene una distribución equilibrada: 50% imágenes reales y 50% imágenes fake.

Diversidad visual

Incluye variedad en iluminación, etnias y tipos de manipulación.

Origen

Proviene de los subconjuntos Deepfakes QA-Patch1 y Patch2.

04 · Obtención de datos

Descarga automatizada.

El dataset se obtuvo de forma automatizada usando Python y kagglehub, asegurando reproducibilidad del flujo de trabajo.

Flujo: Kaggle / Hugging Face → descarga automatizada → carpeta local.

Importancia

Automatizar la descarga permite que el entorno de trabajo cuente con los datos necesarios de forma consistente y reproducible.

05 · Preprocesamiento

Preparando las imágenes.

Antes de entrenar, las imágenes fueron limpiadas, recortadas y ajustadas para que el modelo pudiera procesarlas correctamente.

1. Verificación

Se revisan imágenes corruptas o inválidas.

2. Detección facial

Se usa Haar Cascades para localizar el rostro.

3. Cropping

Se recorta el rostro con margen del 25%.

4. Resize

Se redimensiona a 224×224 px.

5. Modelo

La imagen queda lista para el pipeline de ML.

06 · Transformación a tensores

De imágenes a datos para el modelo.

Después del preprocesamiento, las imágenes se transforman en datos numéricos. Cada píxel contiene valores de color y esos valores se convierten en una matriz procesable.

Representación:
Imagen 224×224 → datos numéricos → tensor.

Normalización

Se ajustan los valores de píxeles usando el procesador del Vision Transformer.

Etiquetas

0 = Fake1 = Real

Serialización

Imágenes y etiquetas se agrupan en un solo archivo .pt.

Eficiencia

Se evita cargar miles de archivos individuales durante el entrenamiento.

09 · Entrenamiento y validación

Preparación para evaluar el modelo.

Para asegurar una evaluación justa, el dataset se dividió en tres conjuntos y se conservó el balanceo de clases.

70%Entrenamiento: aprendizaje de patrones.
15%Validación: ajuste y prevención de sobreajuste.
15%Prueba: evaluación final.

Balanceo de clases

El dataset ya se encontraba balanceado en 50% real y 50% fake, por lo que no fue necesario aplicar oversampling ni undersampling.

15 · Comparación y aporte

¿Qué hace diferente a TrueFace?

Existen enfoques basados en CNN, análisis forense y señales fisiológicas. El problema común es la generalización cuando aparecen nuevos datasets.

Deepfake-o-MeterCombina múltiples modelos.Integra varios detectores.

16 · Limitaciones

Desafíos del proyecto.

Aunque el rendimiento fue alto, el proyecto enfrentó retos de procesamiento y dependencia de recursos computacionales.

Alto costo computacional

El volumen de 50k+ imágenes exige procesamiento intensivo.

Muestreo

Se requiere reducir el dataset para hacerlo manejable.

Hardware potente

El pipeline depende de equipos con buena capacidad de cómputo.

Latencia

El procesamiento de imágenes puede ser un cuello de botella.

TrueFace demuestra que el Machine Learning puede enfrentar desafíos reales de seguridad digital.

Se logró detectar deepfakes con alta precisión (~99.8%). El Vision Transformer permitió capturar patrones complejos en imágenes, y el modelo final es eficiente y aplicable en entornos reales. El proyecto muestra que el ML no solo automatiza procesos, sino que también puede convertirse en una herramienta clave contra la suplantación de identidad.
